Node.js has released its latest security update, version 14.21.3 (LTS), which includes fixes for two CVEs (Common Vulnerabilities and Exposures) that could potentially allow attackers to bypass permissions policies or perform insecure loading of ICU data. The security update also includes OpenSSL security updates and an npm update to address a number of CVEs.

Developers can find more detailed information about each of the vulnerabilities fixed in this release in the February 2023 Security Releases blog post. It is highly recommended that developers update their Node.js installations to the latest version to ensure the security of their applications.

In addition to the security fixes, this release includes several commits, including an update to npm to version 6.14.18, an upgrade of OpenSSL sources to version 1.1.1t, and a build of ICU with ICU_NO_USER_DATA_OVERRIDE. These updates and upgrades aim to improve the performance and stability of Node.js.

For developers who are interested in the technical details of the commits, they can refer to the commit messages for more information. For example, the commit message for the build of ICU with ICU_NO_USER_DATA_OVERRIDE is:

[97a0443f13] - build: build ICU with ICU_NO_USER_DATA_OVERRIDE (RafaelGSS) nodejs-private/node-private#374

This commit aims to address a potential security risk by building ICU with the ICU_NO_USER_DATA_OVERRIDE flag, which prevents the override of user data in ICU data files.

Developers who are using Node.js in their projects should always stay up-to-date with the latest security releases to ensure the security and stability of their applications. Node.js provides long-term support (LTS) releases, which are supported for 30 months, as well as current releases, which are supported for 6 months. It is recommended that developers use LTS releases for production environments to ensure stability and security.

In conclusion, Node.js 14.21.3 (LTS) security release includes important security fixes and updates that improve the performance and stability of Node.js. Developers should update their installations to the latest version to ensure the security of their applications.
