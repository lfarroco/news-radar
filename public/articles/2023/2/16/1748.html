Node.js v18.14.1 (LTS) Security Release

Node.js has released a new version, v18.14.1 (LTS), which includes security updates and bug fixes. This release addresses several vulnerabilities, including CVE-2023-23918, CVE-2023-23919, CVE-2023-23936, CVE-2023-24807, and CVE-2023-23920.

CVE-2023-23918 is a high-severity vulnerability that allows attackers to bypass Node.js permissions policies via process.mainModule. CVE-2023-23919 and CVE-2023-23936 are medium-severity vulnerabilities that affect Node.js OpenSSL error handling and Fetch API, respectively. CVE-2023-24807 is a low-severity vulnerability that can lead to regular expression denial of service in Headers in Node.js fetch API. CVE-2023-23920 is another low-severity vulnerability that involves insecure loading of ICU data through ICU_DATA environment variable.

The release also includes OpenSSL security updates, as outlined in the recent OpenSSL security advisory. Developers are advised to update to this version to avoid potential security risks.

In addition to the security updates, the release includes several bug fixes and commits. Notably, the build process now includes ICU_NO_USER_DATA_OVERRIDE, which helps to prevent user data override in the build process. The crypto library now clears OpenSSL error on invalid ca cert, and Windows ARM64 fix for openssl has been cherry-picked. The undici dependency has also been updated to version 5.19.1.

Developers can find more detailed information on each of the vulnerabilities in the February 2023 Security Releases blog post.

To update to the latest version of Node.js, developers can use the following command:

npm install -g n
n latest

Alternatively, developers can download the latest version from the Node.js website.

It is important for developers to stay up-to-date with the latest security releases and bug fixes to ensure the safety and stability of their applications. With this latest release, Node.js continues to prioritize security and reliability for its users.
