Node.js v16.19.1 (LTS) Security Release

Node.js, the popular JavaScript runtime, has released its latest version v16.19.1 (LTS) with security updates. This release includes fixes for several vulnerabilities, including CVE-2023-23918, CVE-2023-23919, CVE-2023-23920, CVE-2023-23936, and CVE-2023-24807.

CVE-2023-23918 is a high severity vulnerability that allows bypassing of Node.js permissions policies via process.mainModule. CVE-2023-23919 is a medium severity vulnerability that addresses OpenSSL error handling issues in the Node.js crypto library. CVE-2023-23920 is a low severity vulnerability that fixes the insecure loading of ICU data through the ICU_DATA environment variable. CVE-2023-23936 is a medium severity vulnerability that resolves the issue of Fetch API in Node.js not protecting against CRLF injection in host headers. CVE-2023-24807 is a low severity vulnerability that fixes the regular expression denial of service in headers in Node.js fetch API.

The security updates are fixed by an update to undici, a Node.js HTTP/1.1 client and server implementation. For more information on the updates, please refer to the GitHub advisory links provided.

In addition to the security updates, this release includes OpenSSL security updates as outlined in the recent OpenSSL security advisory.

Developers can find more detailed information on each of the vulnerabilities in the February 2023 Security Releases blog post.

To ensure the security of their applications, developers are advised to update to Node.js v16.19.1 (LTS) as soon as possible.
