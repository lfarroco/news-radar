Node v19.6.0 (Current) Release Notes

February 2, 2023 | Topics: Node.js

Node.js, a popular JavaScript runtime, has released version 19.6.0 as the current version. This version comes with several notable changes and updates.

ESM: Leverage loaders when resolving subsequent loaders One of the most significant changes in this release is the ability to apply loaders to subsequent loaders. This means that developers can now use the --experimental-loader flag to specify multiple loaders. For example, --experimental-loader ts-node --experimental-loader loader-written-in-typescript will apply the ts-node loader before the loader-written-in-typescript loader.

Upgrade npm to 9.4.0 Node.js v19.6.0 also comes with an upgraded version of npm, which includes the --install-strategy=linked option for installations similar to pnpm. This option allows developers to install packages in a way that links them to the project, making it easier to manage dependencies.

Other notable changes The release also includes several other notable changes, including the addition of statfs() functions to the fs module, the exposure of cachedDataRejected for vm.compileFunction, support for gc profile in V8, and the addition of the isAscii method to the buffer module.

Developers can also take advantage of the new constrainedMemory API for the process module and the addition of reporters to the test runner.

Overall, Node.js v19.6.0 brings several enhancements and improvements that will make it easier for developers to work with the platform. As always, developers are encouraged to upgrade to the latest version to take advantage of the latest features and improvements.

// Example of using multiple loaders with ESM
import { createRequire } from 'module';
const require = createRequire(import.meta.url, {
  loader: {
    '.ts': 'ts-node',
    '.tsx': 'ts-node',
  }
});

console.log(buffer.isAscii()); // true
