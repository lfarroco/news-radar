Node.js, a popular JavaScript runtime, has just released its latest version, v20.1.0. This release includes several changes and improvements that will benefit developers who use Node.js in their projects.

Notable Changes One of the notable changes in this release is the deprecation of CallTracker in the assert module. This change was made to improve the performance of the module. Additionally, the crypto module has been updated to use the latest root certificates from NSS 3.89.

Another significant change is the addition of the getDefaultResultOrder method to the dns module. This method allows developers to specify the order in which DNS query results are returned.

The fs module has also received some updates in this release. The readdir and opendir methods now have a recursive option, which makes it easier to recursively read directories. The cp methods now support a mode flag to specify the copy behavior.

The http module has a new highWaterMark option for the createServer method. This option allows developers to set the high water mark for the server's internal buffer.

The stream module has also received some updates. The compose method now preserves object mode, and the test_runner module now has a new API for specifying test name patterns and executing before hooks.

Finally, the wasi module now has the returnOnExit option set to true by default, which makes it easier to handle exit codes.

Commits In addition to the notable changes, this release includes several commits that improve the performance and stability of Node.js. One of the commits adds a new benchmark for eventtarget creation, while another differentiates between whatwg and legacy URLs.

Conclusion Node.js v20.1.0 includes several changes and improvements that will benefit developers who use Node.js in their projects. From performance improvements to new features in various modules, this release is worth checking out for anyone who wants to keep up with the latest news in the Node.js ecosystem.
