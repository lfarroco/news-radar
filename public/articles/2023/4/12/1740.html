Node.js v19.9.0: Tracing Channel and New URL.canParse API

April 12, 2023 | Topics: Node.js

Node.js has released its latest version, v19.9.0, with some notable changes that will benefit developers. The new version includes a Tracing Channel in diagnostic_channel, which adds a high-performance channel to publish tracing data about the timing and purpose of function executions. This feature was contributed by Stephen Belanger in #44943.

Another noteworthy addition is the new URL.canParse API, which checks if an input with an optional base value can be parsed correctly according to the WHATWG URL specification. This new API was contributed by Khafra in #47179. Here's an example of how to use it:

const isValid = URL.canParse('/foo', 'https://example.org/'); // true
const isNotValid = URL.canParse('/foo'); // false

In addition to these changes, there are other notable updates in this version, such as:

  • The getMaxListeners method was added to the events module by Khafra in #47039.
  • The msi module was migrated to WiX4 by Stefan Stojanovic in #45943.
  • The node-api module has deprecated the napi_module_register method by Vladimir Morozov in #46319.
  • The stream module has added a setter and getter for default highWaterMark by Robert Nagy in #46929.
  • The test_runner module has exposed a reporter for use in the run API by Chemi Atlow in #47238.

If you're interested in the commits, there were two notable ones:

  • Deokjin Kim fixed an invalid requirements URL in the benchmark module in #47378.
  • Khafra lowered the number of URL.canParse runs in the benchmark module in #47351.

Node.js is a popular runtime environment for executing JavaScript code outside of a web browser. It is built on top of the V8 JavaScript engine and provides a powerful set of features for server-side development. With the new changes in v19.9.0, developers can take advantage of the Tracing Channel and the new URL.canParse API to improve the performance and reliability of their applications.
