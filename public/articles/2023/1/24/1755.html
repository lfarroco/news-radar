Node.js v19.5.0 (Current) Release Notes

Node.js, a popular JavaScript runtime, has released its latest version, v19.5.0 (Current). This release includes several notable changes and bug fixes.

Notable Changes One of the notable changes in this release is the addition of webstreams to Duplex.from() in the lib:: module. This change was made by Debadree Chatterjee and is available in issue #46190. Additionally, the stream:: module now implements finished() for ReadableStream and WritableStream, also by Debadree Chatterjee, in issue #46205.

Another notable change is the addition of join authorization headers in the http: module, available in issue #45982, made by Marco Ippolito.

Commits This release includes several commits, including removing assert.snapshot in the assert module by Moshe Atlow in issue #46112, using os.availableParallelism() in the benchmark and tools modules by Deokjin Kim in issue #46003, and adding extra semi check in the build module by Jiawen Geng in issue #46194.

In the crypto module, there were several commits, including adding CryptoKey Symbol.toStringTag by Filip Skokan in issue #46042, adding cipher update/final methods encoding validation by vitpavlenko in issue #45990, and ensuring auth tag set for chacha20-poly1305 by Ben Noordhuis in issue #46185.

Other commits include updating simdutf to 3.1.0 in the deps module by Node.js GitHub Bot in issue #46257, bumping googletest to 2023.01.13 in the deps module by Jiawen Geng in issue #46198, and adding .gitignore files in the deps module by various contributors in issue #46262.

Developers can check out the full list of commits and changes in the official release notes.

Conclusion Node.js v19.5.0 (Current) includes several notable changes and bug fixes, making it a recommended update for developers. With the addition of webstreams to Duplex.from() and finished() for ReadableStream and WritableStream, developers can expect improved functionality in their projects. Additionally, the various commits and bug fixes in this release ensure a more stable and efficient runtime for JavaScript development.
